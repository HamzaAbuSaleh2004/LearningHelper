Kafka partition rebalancing is a critical concept. It happens when a new consumer joins a group, an existing consumer leaves, or a new partition is added. Think of it as redistributing workloads.

// Trigger Events for Rebalance
1. Consumer Group membership changes.
2. Topic metadata changes (partitions added).

// Important Note:
During rebalance, consumers cannot consume messages. This is known as a "Stop the World" event in older Kafka versions.
